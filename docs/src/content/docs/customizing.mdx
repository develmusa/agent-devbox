Extend Agent Devbox for additional languages, domains, or tools.

## Add Language Support

Edit `.devcontainer/Dockerfile` and uncomment the sections you need:

### Python

```dockerfile
RUN apt-get update && apt-get install -y \
    python3 python3-pip python3-venv python3-dev build-essential \
    && pip3 install --no-cache-dir --break-system-packages \
        uv ruff black mypy pytest ipython
```

### Go

```dockerfile
COPY --from=golang:1.22 /usr/local/go /usr/local/go
ENV PATH="/usr/local/go/bin:${PATH}"
```

### Rust

```dockerfile
RUN curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
ENV PATH="/root/.cargo/bin:${PATH}"
```

After editing, rebuild: `F1 → "Dev Containers: Rebuild Container"`

## Whitelist Domains

Edit `.devcontainer/scripts/init-firewall.sh`:

```bash
ALLOWED_DOMAINS=(
    # ... existing domains ...
    "your-domain.com"
)
```

**Guidelines:**
- Justify each addition
- Use official domains (not CDNs)
- Avoid wildcards
- Test after adding

## Add VS Code Extensions

Edit `.devcontainer/devcontainer.json`:

```json
"extensions": [
  "eamodio.gitlens",
  "usernamehw.errorlens",
  "your-extension-id"
]
```

## Modify Resource Limits

Edit `.devcontainer/devcontainer.json`:

```json
"runArgs": [
  "--memory=8g",     // Increase RAM
  "--cpus=4"         // More CPU cores
]
```

## Project Structure

```
.devcontainer/
├── devcontainer.json    # Container configuration
├── Dockerfile           # Base image
├── scripts/
│   ├── init-firewall.sh # Network security
│   └── post-create.sh   # Dependency installer
├── README.md            # User guide
└── SECURITY.md          # Threat model
```

## Best Practices

1. **Commit frequently** — Track all changes
2. **Test firewall** — Verify domains work after whitelisting
3. **Keep minimal** — Only add what you need
4. **Document changes** — Comment why domains were added
